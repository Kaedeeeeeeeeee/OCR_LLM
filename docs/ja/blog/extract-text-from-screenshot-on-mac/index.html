Mac でスクリーンショットからテキストを抽出する方法（3 つの方法を比較）

2026 年 4 月 25 日 · 10 分で読める

要約 2026 年現在、Mac のスクリーンショットからテキストを抽出する確実な方法は 3 つあります。プレビューで開いて Live Text を使う、Cmd+Shift+5 で撮ってフローティングサムネイル上で Live Text を使う（macOS 13 以降）、そして専用 OCR ホットキーでファイル化のステップ自体を省略する。前者 2 つは無料で標準機能、3 つ目はもっと速くて、1 ドラッグでテキストがクリップボードに入ります — Cheese! OCR が組み立てているのは、まさにこの導線です。

Mac を使っていれば、週に何度か同じ場面に出くわします。Slack で送られてきたエラーメッセージのスクリーンショット、LINE で共有された予約確認画面、Zoom 会議で共有された資料の一部、駅看板を撮った写真 — その中の「テキストだけ」が欲しい。けれど、Notion 上のテキストのようには選択できません。ピクセルはピクセルだからです。

幸いなことに、macOS 12 Monterey で Live Text が登場して以降、Mac でのスクリーンショット OCR はずいぶん簡単になりました。ただ少し残念なのは、組み込みの導線では 1 日に 20 回も使うとどうしても手数が多く感じる点と、多くの人が知らない Cmd+Shift+5 のフローティングプレビューを使えば 1〜2 ステップ短縮できるという点です。そしてそれを業務で日常的に使う人 — エンジニアがスタックトレースを取り出す、サポート担当が顧客スクリーンショットから情報を引っ張る、論文 PDF のスクショから引用文を抜く学生 — のためには、3 つ目の方法があり、これは全工程をほぼ「1 回のドラッグ」に圧縮します。

この記事では 3 つの方法をそれぞれ説明し、横並びで正直に比較し、どの場面でどれを使うべきかをお伝えします。

まずは Mac の 3 つのスクリーンショットショートカットの復習

テキスト抽出に入る前に、3 つの標準ショートカットは覚えてしまった方が後がラクです。どの方法でも前提として使います。

  • Cmd+Shift+3 — 画面全体（接続中のすべてのディスプレイ）をデスクトップに即保存。
  • Cmd+Shift+4 — 十字カーソルになり、領域をドラッグで指定。ドラッグ中に Space を押すとウインドウキャプチャモードに切り替わり、好きなウインドウをクリックすればドロップシャドウ込みできれいに撮れます。
  • Cmd+Shift+5 — フローティングのキャプチャツールバーが出ます。フルスクリーン、ウインドウ、領域、画面収録、そしてオプションメニュー付き。これが現行の中心ショートカットで、方法 2 はここに住んでいます。

3 つとも標準では スクリーンショット YYYY-MM-DD HH.MM.SS.png という名前の PNG をデスクトップに保存します。保存先は Cmd+Shift+5 → オプション → 保存先 で変更できます。この設定はあとで何度か出てきます。どの方法が手に馴染むかが、この設定で実際に変わってくるからです。

もう 1 つ知っておくと役立つのは、最近の macOS（Mojave 以降）でスクリーンショットを撮ると、画面右下にサムネイルが数秒間ふわっと浮かぶ点です。このサムネイルは思っているより働き者で、方法 2 ではこれが主役になります。

方法 1：スクリーンショット → プレビュー → Live Text

最も発見しやすい導線です。Apple の公式ドキュメントが指し示しているのもこれで、多くの人が試行錯誤で辿り着くのもこの方法です。

  1. Cmd+Shift+4（フルスクリーンなら Cmd+Shift+3）を押す。
  2. 領域をドラッグ。スクリーンショットがデスクトップに保存される。
  3. ファイルをダブルクリック。標準ではプレビュー（Preview）で開く。
  4. プレビュー内で文字の上をドラッグ。文字の上にカーソルを持っていくと I ビームに変わり、普通の文書と同じように選択できる。
  5. Cmd+C でコピー。

独立した操作が約 5 ステップ。そしてデスクトップにはスクリーンショットファイルが残り続けます（あとで掃除するまで）。

プレビューの Live Text は本当によくできています。背後で動いているのは Apple Vision の同じパイプラインで、macOS 全体の OCR と同じ系譜です。英語は安定、日本語も縦書きを含めて十分実用、中国語・韓国語、フランス語・ドイツ語・スペイン語・イタリア語・ポルトガル語・ウクライナ語まで認識します。手書きもきれいなものなら拾えます。ダークモード UI のスクリーンショットでも問題なし。データ検出も効くので、電話番号や URL は右クリックで電話アプリやブラウザに渡せます。

不満点は実質 2 つ。手数（5 ステップ）と、デスクトップにファイルが溜まる問題です。週に 1 度くらいなら気にならないコストですが、毎日となると地味に効いてきます。

方法 1 がぴったりな場面

  • スクリーンショットがすでに手元にある — 誰かから送られてきた、以前自分で撮った。
  • テキストだけでなく画像自体も残しておきたい。
  • 取り出す前にマークアップやトリミングをしたい。
  • 自分のものではない Mac を借りていて、アプリを入れられない。

方法 2：Cmd+Shift+5 フローティングプレビュー + Live Text

これは過小評価されているショートカットです。スクリーンショット後に右下に出るフローティングサムネイルは多くの人が知っていますが、無視するか、Markup を開くためだけに使うかのどちらかです。macOS 13 Ventura 以降では、そのサムネイル上で直接 Live Text が使えて、プレビューを開かずに済むことを知っている人は意外と少ないのです。

導線は次の通りです。

  1. Cmd+Shift+5 を押す。
  2. キャプチャモードを選ぶ（領域キャプチャが多いはず）。
  3. 領域をドラッグ。
  4. 右下にサムネイルが浮かぶ。消える前にクリック（数秒の猶予あり、もう 1 回クリックで固定もできる）。
  5. Markup ウインドウが開く。文字の上でカーソルが I ビームに変わるのでドラッグして選択、Cmd+C。

同じく約 5 ステップですが、設定を整えておけばファイルは 1 つも作られません。Cmd+Shift+5 → オプション → 保存先：クリップボード にすると、画像はディスクに書かれずクリップボードへ直接行きます。こうするとフローティングプレビューは実質「一時的な注釈レイヤー」になります — テキストをコピーし終えてウインドウを閉じれば、何も残りません。

オプションでもう 1 つ便利なのがフローティングプレビューの表示時間です。デフォルトでは数秒で消えますが、明示的に長さを伸ばすスライダーはありません。代わりに 1 回クリックすれば固定できます。サムネイルを取り逃しがちな人は、右下に視線を予約しておく癖をつけると安定します。

方法 2 が勝つ場面

  • スクリーンショットを頻繁に撮るが、デスクトップを散らかしたくない。
  • コピー前に簡単な注釈を入れることがある — Markup がそこにある。
  • OS が macOS 13 Ventura 以降（Markup 内 Live Text は Ventura 以降が要件）。

注意点

サムネイルは消える前にクリックする必要があります。そして Markup 内でも自分でドラッグして選択しなければ取り出せません — 「画像内のすべてのテキストをまとめて」というボタンはありません。1 枚に 10 行ばらばらに書かれた文字すべてが必要なら、10 行分のドラッグが要ります。文字の多いスクリーンショットでは少し疲れます。

方法 3：専用 OCR ホットキー（Cheese! OCR）

これは私たちが作っているプロダクトなので、その点は割り引いて読んでください。それでもワークフロー自体は事実として書けます。

  1. ⇧⌘E（または自分で設定したホットキー）を押す。
  2. テキストを含む領域をドラッグ。

これだけです。ファイルは作られません。ウインドウも開きません。認識済みのテキストはすでにクリップボード上にあり、どこにでも貼り付けられます。ドラッグ終了地点の近くに小さな確認チップが一瞬出て、結果はメニューバー内の検索可能な履歴にも保存されます。

これが方法 1・2 より速い理由は、OCR 本体ではありません。Cheese! OCR は Live Text と同じ Apple Vision フレームワークを使っているので、認識にかかる時間はほぼ同じです。速いのは「テキストが欲しい」と「テキストがクリップボードに入った」の間の工程が圧縮されているからです。ファイル保存なし、アプリ切り替えなし、Markup ウインドウなし、撮った画像内での再ドラッグなし。スクリーンショット領域を決めるドラッグが、そのまま OCR 領域を決めるドラッグになっています。

業務として日常的に使う場合、もう少し細かい点も効いてきます。

  • 検索可能な履歴。OCR の結果は元画像のサムネイル付きでローカルに保存されます。「先週どこかでエラーメッセージを OCR したな」という記憶を頼りに、数週間後でも検索で取り戻せます。
  • 多言語のデフォルト対応。英語・簡体字中国語・日本語・韓国語をモード切替なしで認識します。混在（英語 UI に日本語のエラー、Slack スレッドにコードスニペット）も透過的に処理されます。
  • 100% オンデバイス。Cheese! OCR にはネットワーク権限が一切なく、App Store サンドボックスで検証可能です。スクリーンショットの内容は Mac から外に出ません。社内ダッシュボード、契約書、顧客データなど機微な内容を撮る人にとって、これは実質的な差です。
  • 買い切り。$5.99 の一度きり。サブスクなし。Mac App Store がレシートとアップデートを管理します。

正直なトレードオフ

もう 1 つアプリを入れ、もう 1 つホットキーを設定し、$5.99 を払うことになります。スクリーンショット OCR の頻度が低い人 — 週 1 回・月 1 回 — なら、方法 1・2 で十分ですし、無料です。境目はだいたい「1 日に 1 回以上」のあたり。この頻度になると、節約される秒数と検索可能な履歴が、実際の時間として積み上がってきます。

それぞれの方法はいつ使うべきか

友人に聞かれた時に答える、実用的な目安です。

方法 1（プレビュー + Live Text）が向く場合

  • すでに画像ファイルが手元にある — AirDrop や Slack で受け取った、先週保存した、メールの添付。
  • テキストを取り出す前にマークアップやトリミングをしたい。
  • 借りものの Mac を一時的に使っている。
  • 頻度がそもそも低い。

方法 2（Cmd+Shift+5 フローティングプレビュー）が向く場合

  • 標準機能でいきたいが、デスクトップを散らかしたくない。「保存先：クリップボード」を一度設定するだけで摩擦がぐっと減ります。
  • コピー前に簡単な矢印や枠を足すことがある — Markup がそこにある。
  • OS が macOS 13 Ventura 以降。

方法 3（Cheese! OCR のような専用ツール）が向く場合

  • スクリーンショット OCR を 1 日に 2 回以上行う。
  • 過去の認識結果をよく検索する（「先週見たチュートリアルから OCR したコマンド、何だったっけ？」）。
  • 機微な内容を扱うので、ネットワーク権限ゼロという検証可能な保証が欲しい。
  • 言語が混ざる場面が多い — Slack の英日混在、Zoom の日本語資料、漫画のセリフ、駅看板。
  • デスクトップに スクリーンショット 2026-04-25.png が並ぶのにそろそろ嫌気がさしている。

3 つの方法すべてに共通するパワーユーザー向けの小ワザ

「保存先」をクリップボードにする

意外に紹介されない、最もコスパの良い macOS スクリーンショット改造です。Cmd+Shift+5 → オプション → 保存先 → クリップボード。これ以降、すべてのスクリーンショットはクリップボードに入り、デスクトップにファイルが落ちなくなります。Slack、Notion、メール、どこにでも直接貼り付けられます。逆に「今回はファイルが欲しい」というときは Control を押しながら撮れば一時的に挙動を反転できます（現在の設定の逆方向）。

サムネイルを取り逃すなら、ピン留めする

右下に出てきたら 1 回クリックして固定します。連続でいくつか撮るときに特に便利です。

Cmd+Shift+4 → Space でウインドウをきれいに撮る

多くの人はウインドウの周囲をドラッグで囲んで撮りますが、これだと背景まで写ります。Cmd+Shift+4 → Space → 対象ウインドウをクリックすれば、角丸とドロップシャドウを保ったまま、背景なしで撮れます。シャドウが要らない場合は Option を押しながらクリック。

Cmd+Shift+5 ツールバーからも同じ操作ができる

「選択したウインドウを取り込む」がフローティングコントロールにあります。ショートカットを忘れたときの保険に。

Cheese! OCR 限定：履歴検索で過去の結果を呼び出す

同じ種類の領域（業務ダッシュボードの注文番号など）を繰り返し OCR するなら、検索可能な履歴がそのまま「過去の抽出結果のタイムライン」として機能します。メニューバーウインドウを開いて Cmd+F で絞り込めば、数日前の結果も数秒で取り出せます。

Cheese! OCR 限定：既存画像を渡す

メールや AirDrop で受け取ったスクリーンショットは撮り直す必要はありません。Cheese! OCR のメニューバーウインドウに画像をドロップするか、クリップボードからペーストすれば、同じ流れで認識できます。撮影自体が方法 3 でなくても、抽出のステップは方法 3 で済ませられます。

ホットキーは自分の指に合わせる

⇧⌘E が指に合わない場合（F キーが好き、衝突しにくい ⌃⌥⌘ 系の組み合わせが好き）、Cheese! OCR の設定で変更できます。同じことが macOS 標準にも言えます — 「システム設定 → キーボード → キーボードショートカット → スクリーンショット」で Cmd+Shift+3/4/5 をリマップできるので、他のツールと衝突したら遠慮なく差し替えてください。

3 つの方法のどれかが「正しい」というわけではなく、それぞれ違う頻度・違うワークフローに最適化されているだけです。週に 1 回、同僚と共有するためにスクリーンショットを撮り、たまにそこから 2 単語を抜き出したい、くらいなら標準機能で十分です。一方、テキスト抽出が仕事の一部になっている場合 — 講義スライドのスクショから引用を抜く、Slack スレッドからエラーメッセージを取り出す、ダッシュボードのスクリーンショットから 1 日に何度も数値をコピーする — なら、専用ホットキーは目に見えて差を生みます。まずは無料の方法から試してみて、「この数キーが惜しい」と思った瞬間が、Cheese! OCR のようなツールが元を取り始めるタイミングです。

よくある質問

Mac のスクリーンショットはどこに保存されますか？
デフォルトでは PNG ファイルとしてデスクトップに保存され、ファイル名は日付と時刻になります。保存先を変えるには、Cmd+Shift+5 を押してオプションをクリックします。プルダウンからデスクトップ、書類、クリップボード、メール、メッセージ、プレビュー、または任意のフォルダを選べます。
ファイルを作らずにスクリーンショットを撮ることはできますか？
できます。Cmd+Shift+5 を押し、オプション → 保存先：クリップボード を選びます。画像はディスクに書き込まれずクリップボードに直接入ります。Cheese! OCR のような専用 OCR ホットキーツールはさらに一歩進んでいて、画像ファイルすら作らず、認識済みのテキストをそのままクリップボードに渡します。
Cmd+Shift+5 の方法は古い macOS でも動きますか？
Cmd+Shift+5 のフローティングキャプチャツールバーは 2018 年の macOS Mojave (10.14) から存在するので、現役で使っている Mac ならほぼ全機種で利用できます。ただしフローティングサムネイル内で Live Text を使うには macOS 13 Ventura 以降が必要です。それより古い OS でもキャプチャは可能ですが、ファイルをプレビューで開いてから Live Text を使う流れになります。
3 つの方法のうち、テキスト抽出が最も速いのはどれですか？
方法 3、つまり Cheese! OCR のような専用 OCR ホットキーです。ホットキー 1 回 + ドラッグ 1 回で、認識済みのテキストはもうクリップボードにあります。ファイルは作られず、プレビューを開く手間もありません。Live Text なら最低 2 ステップ追加（Live Text ボタンを押してコピー）、Cmd+Shift+5 のフローティングプレビューも同程度の手数になります。
すでに保存済みのスクリーンショットでも Cheese! OCR は使えますか？
使えます。既存の画像を Cheese! OCR のメニューバーウィンドウや OCR 履歴ペインにドロップすれば再認識できます。メインの導線はグローバルホットキーですが、すでに保存された PNG や JPEG も問題なく扱えます。
